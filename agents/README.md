# GenesisAI: Стартовый репозиторий для разработки, управляемой ИИ

## Введение

Добро пожаловать в **GenesisAI** — универсальный стартовый репозиторий, разработанный для ускорения процесса создания любого проекта. `GenesisAI` представляет собой инновационный подход к разработке, где интеллектуальные агенты ИИ берут на себя рутинные, но критически важные задачи: от создания полной и точной документации до генерации рабочего кода, строго соответствующего спецификациям.

Начните свой следующий проект, сосредоточившись на идее, а не на начальной рутине.

## Концепция GenesisAI

`GenesisAI` создан как основа, с которой можно начать любой проект, независимо от используемых технологий (Flutter, ReactJS, Angular, Python, Go и др.). Ключевая идея заключается в делегировании первичной разработки агентам:

1.  **Документация и Спецификации**: Агенты ИИ анализируют высокоуровневые требования и генерируют подробную документацию, user-flow, спецификации для фронтенда и бэкенда, а также схемы баз данных. Эта документация является основой для дальнейшей разработки.
2.  **Генерация Кода**: После утверждения документации другие агенты ИИ используют эти спецификации для написания чистого, поддерживаемого и полностью функционального кода. (Это находится в стадии активной разработки и планируется к внедрению).

Цель `GenesisAI` — обеспечить быструю и стандартизированную отправную точку для любого проекта, минимизируя усилия на начальных этапах.

## Начало работы

Чтобы начать новый проект с `GenesisAI`, выполните следующие шаги:

1.  **Клонирование репозитория**:
    ```bash
    git clone https://github.com/bessonniy11/GenesisAI.git
    cd GenesisAI
    ```
2.  **Запуск агентов для генерации документации**:
    В папке `agents` находятся скрипты для управления агентами. Используйте их для запуска процесса генерации документации на основе ваших первоначальных требований.
    ```bash
    # Пример команды для запуска оркестратора агентов документации (уточните по мере развития)
    node agents/doc_agents/orchestrator/doc-agent-orchestrator.js
    ```
    *Примечание*: После запуска агенты создадут или обновят файлы документации в соответствующих папках (например, `docs/`).

3.  **Проверка и утверждение документации**:
    Внимательно изучите сгенерированную документацию. Внесите необходимые корректировки или дайте обратную связь агентам для улучшения.

4.  **Генерация кода (планируется)**:
    После утверждения документации, будущие версии `GenesisAI` позволят запускать агентов для генерации самого кода проекта на основе утвержденных спецификаций.

## Структура проекта (ожидаемая)

Когда агенты начнут работу, они будут создавать следующую структуру (или адаптировать существующую):

*   `agents/`: Содержит все скрипты и конфигурации для работы агентов ИИ.
    *   `doc_agents/`: Агенты, отвечающие за генерацию документации.
    *   `code_agents/`: (Планируется) Агенты, отвечающие за генерацию кода.
*   `docs/`: Папка для всей сгенерированной документации.
    *   `project_documentation/`: Общая документация по проекту.
    *   `jira/`: Документация, разбитая по задачам (например, `auth/`, `backend-admin-panel/`).
        *   `spec-front.md`: Спецификации для фронтенда.
        *   `spec-backend.md`: Спецификации для бэкенда.
        *   `user-flow.md`: Описание пользовательских сценариев.
        *   `db-setup.md`: Схемы баз данных и инструкции по их настройке.
        *   `tasks.md`: Список задач и подзадач для проекта.
*   `src/` или другие папки, специфичные для технологий: Здесь будет генерироваться код проекта (например, `lib/` для Flutter, `frontend/`, `backend/`).
*   `README.md`: Основной README для всего проекта.

## Разработка с агентами

Процесс разработки с `GenesisAI` будет выглядеть так:

1.  **Определите высокоуровневые требования**: Опишите, что вы хотите получить от проекта.
2.  **Запустите агентов документации**: Позвольте им создать подробные спецификации.
3.  **Итерируйте с документацией**: Уточняйте и дорабатывайте документацию с помощью агентов до полного соответствия вашим ожиданиям.
4.  **Запустите агентов кода (будущее)**: После утверждения документации, агенты сгенерируют код.
5.  **Финальный рефакторинг и тестирование**: Несмотря на автоматизацию, человеческое участие в финальном рефакторинге и тестировании всегда будет ценным.

## Как это работает (кратко)

Агенты ИИ используют контекст всего репозитория, предоставленные инструкции и текущее состояние проекта для выполнения своих задач. Они читают файлы, анализируют структуру, генерируют новый контент и, при необходимости, вносят изменения, действуя как высококвалифицированные члены команды разработчиков.
